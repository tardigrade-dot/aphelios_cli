# 流式播放卡顿问题修复说明

## 问题原因

你之前遇到的"合成 - 播放 - 合成 - 播放"卡顿问题是因为：

1. **播放器等待每个 chunk 完成**：之前的代码使用 `sink.sleep_until_end()` 等待每个音频块播放完才接收下一个
2. **同步阻塞**：合成线程和播放线程之间是串行关系，不是并行

## 修复方案

### 修改 1: 真正的异步播放器

**之前的问题代码：**
```rust
// 错误：等待每个 chunk 播放完
sink.append(source);
sink.sleep_until_end();  // ❌ 阻塞在这里
```

**修复后的代码：**
```rust
// 正确：立即追加，不等待
sink.append(source);  // ✅ 立即返回，继续接收下一个 chunk
// 只在最后等待一次
sink.sleep_until_end();
```

### 修改 2: 减小 chunk 大小

```rust
SynthesisOptions {
    chunk_frames: 5,  // 从 10 减到 5 (400ms)
    // 更小的 chunk = 更及时的播放
}
```

### 修改 3: 降低预缓冲时间

```rust
let target_buffer_secs = 1.0;  // 从 2.0 秒减到 1 秒
// 更少的预缓冲 = 更快开始播放
```

## 修复后的行为

```

0.0s  ┌─ 合成开始
      │
1.5s  ├─ 第 1 块合成完成 → 发送到播放器
      │                   ├─ 预缓冲 (1 秒)
2.5s  ├─ 第 2 块合成完成 → 追加到播放器
      │                   ├─ 播放开始 ✅
3.0s  ├─ 第 3 块合成完成 → 追加到播放器
      │                   │  播放中...
3.5s  ├─ 第 4 块合成完成 → 追加到播放器
      │                   │  播放中...
      ...
```

**关键改进：**
- ✅ 合成和播放**完全并行**
- ✅ 播放不等待合成（除了最初的 1 秒预缓冲）
- ✅ 没有"合成 - 播放 - 合成 - 播放"的卡顿

## 技术细节

### 播放器架构

```
合成线程                    播放线程
    │                          │
    ├── 合成第 1 块 ────────────→│ 追加到队列
    │                          │ 预缓冲中...
    ├── 合成第 2 块 ────────────→│ 追加到队列
    │                          │ 开始播放 ✅
    ├── 合成第 3 块 ────────────→│ 追加到队列
    │                          │ 继续播放
    ...                        │
    ├── 合成完成 ──────────────→│ 等待播放结束
                               │ 播放完成
```

### 关键代码

```rust
// player.rs - 播放循环
fn playback_loop(...) {
    // 1. 预缓冲阶段
    while buffered < target {
        sink.append(source);  // 追加，不等待
    }
    
    // 2. 连续播放阶段
    while let Ok(chunk) = recv() {
        sink.append(source);  // 追加，不等待 ✅
    }
    
    // 3. 最后等待一次
    sink.sleep_until_end();
}
```
